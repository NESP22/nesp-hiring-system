# NESP Custom Domain And Off-Platform Backup Plan

This document records the protected-review setup for the NESP OpenCATS hiring
portal. It does not authorize public launch, removal of Basic Auth, outbound
email, external job-board posting, real applicant collection, or unrelated DNS
changes.

- Service: `nesp-hiring-web`
- Temporary URL: `https://nesp-hiring-web.onrender.com`
- Protected review hostname: `careers.nesportsphoto.com`

| Type | Name / Host | Target |
| --- | --- | --- |
| `CNAME` | `careers` | `nesp-hiring-web.onrender.com` |

DNS guardrails:

- Add only the `careers` subdomain record.
- Do not modify the root domain, `www`, MX, TXT, SPF, DKIM, DMARC, or unrelated

  issues TLS unless Render or Craig explicitly approves a proxy change.

## Encrypted Off-Platform Backup Plan

The Render on-disk backup confirms backup generation works, but it is not enough
for launch because the backup remains on the same hosting platform. Before
accepting real applicants, perform an encrypted off-platform backup and restore
test.

Approved destination requirements:

- Craig-approved storage location, such as an encrypted company cloud drive,
  private S3-compatible bucket, or encrypted local/offsite storage.
- Access limited to Craig and explicitly approved NESP administrators.
- Encryption before upload or transfer.
- No public links.
- No backup files committed to git.
- No applicant uploads or database dumps copied to unapproved third-party
  services.

Approved test policy:

- Backup owner: Craig Edwards.
- Recovery process owner: Craig Edwards, with Codex-assisted documented
  restore steps.
- Retention: keep the latest 7 daily encrypted backups and 4 weekly encrypted
  backups.
- Test destination type: Craig-controlled local Mac storage outside the
  repository.

Suggested backup package:

- Latest `database.sql.gz` generated by `nesp-render-backup`
- Latest `opencats-files.tar.gz`
- Backup `README.txt`
- A manifest with timestamp, source service, file sizes, and SHA-256 checksums

Suggested encryption:

- Use age, GPG, or an equivalent approved encryption tool.
- Store private keys outside the repository and outside Render.
- Record the recovery owner and key location in an internal NESP-only document.

Restore test procedure:

1. Create a fresh backup with `nesp-render-backup`.
2. Copy only the generated backup archive files to the approved destination.
3. Download the encrypted backup to a separate local or temporary test
   environment.
4. Decrypt locally.
5. Restore the database dump into a disposable MariaDB instance.
6. Restore OpenCATS persistent files into a disposable application directory.
7. Confirm the five NESP jobs and the fictional `Test Applicant` record restore.
8. Confirm no outbound mail credentials or enabled mail settings appear in the
   restored environment.
9. Delete the disposable restored environment and decrypted local files after
   verification.

Launch gate:

- Public launch is blocked until one encrypted off-platform backup and restore
  test succeeds and Craig approves the backup owner, retention period, and
  recovery process.
- The 2026-07-13 test record is documented in
  `docs/nesp/BACKUP_RESTORE_TEST_2026-07-13.md`. Backup encryption,
  off-platform transfer, checksum verification, database restore, file restore,
  and job verification succeeded, but the launch gate remains blocked pending
  Craig's decision on the missing fictional Test Applicant verification.
